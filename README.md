Профессиональный Telegram-бот для автоматизации записи пациентов к специалистам. Позволяет клиентам выбирать услугу, специалиста, дату и время визита, а также автоматически отправляет напоминания за 24 часа. Для администраторов предусмотрена панель управления расписанием и каталогом услуг.

---

## ✨ Основной функционал

### 👤 Для клиентов
* **Бесшовная запись (FSM):** Пошаговый выбор услуги → специалиста → даты (удобный Inline-календарь) → времени.
* **Личный кабинет (`/my_appointments`):** Просмотр предстоящих записей с возможностью отмены (не позднее чем за X часов).
* **Умные уведомления:** Автоматические напоминания о визите за 24 часа.
* **Конфиденциальность:** Регистрация по номеру телефона (через нативный Telegram Contact Sharing).

### 🛡 Для администраторов (`/admin`)
* **Управление записями:** Просмотр списка (с фильтрацией), перенос записей на другое время (с уведомлением клиента), отмена с указанием причины.
* **Каталог услуг (CRUD):** Создание, редактирование и деактивация услуг.
* **Управление специалистами:** Добавление врачей, настройка расписания (с поддержкой выходных дней).

## 🏗 Архитектура и технологии

Проект спроектирован с учетом **чистой архитектуры** и лучших практик разработки (SOLID, Dependency Injection):
- **Стек:** `aiogram 3.x`, `SQLAlchemy 2.0 (Async)`, `aiosqlite`.
- **База данных:** SQLite (для простоты деплоя) с использованием `Alembic` для управления миграциями.
- **Бизнес-логика:** Вынесена в независимый слой `services/` (генерация слотов, управление записями), что позволяет легко покрывать код Unit-тестами (`pytest`).
- **Транзакционность:** Строгий контроль гонок (Race Conditions) при бронировании — защита от двойной записи на одно время.
- **Фоновые задачи:** `APScheduler` (сохранение Job'ов в БД через SQLAlchemyJobStore) гарантирует, что запланированные напоминания не пропадут при перезапуске сервера.
- **Качество кода:** Код отформатирован и проверен линтером `Ruff`.

## 🚀 Инструкция по запуску

Проект полностью готов к Production-развертыванию через Docker.

### 1. Подготовка окружения
Склонируйте репозиторий и создайте файл конфигурации:
```bash
git clone https://github.com/your-username/tg-appointment-bot.git
cd tg-appointment-bot
cp bot/.env.example bot/.env
```

Отредактируйте `bot/.env`, добавив ваш `BOT_TOKEN` (от @BotFather) и ваш `ADMIN_IDS` (ваш Telegram ID).

### 2. Запуск через Docker (Рекомендуется)
```bash
docker-compose up -d --build
```
Бот и база данных будут автоматически инициализированы. При первом запуске скрипт `init_db.py` создаст демо-специалистов и расписание.

### 3. Запуск локально (для разработки)
```bash
python -m venv .venv
source .venv/bin/activate  # Или .venv\Scripts\activate на Windows
pip install -r requirements.txt

cd bot
# Применение миграций Alembic
alembic upgrade head

# Инициализация демо-данных (опционально)
python init_db.py

# Запуск бота
python bot.py
```

## 🧪 Тестирование
В проекте настроен `pytest`. Для запуска базовых юнит-тестов выполните:
```bash
pytest
```

## 🔄 Возможности масштабирования
Архитектура позволяет легко заменить SQLite на PostgreSQL (достаточно изменить `DATABASE_URL` и драйвер SQLAlchemy на `asyncpg`), а также вынести APScheduler в отдельный процесс (Celery/Redis) при высоких нагрузках.
